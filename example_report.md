## Manual Testing Checklist

For each high-interest endpoint above, consider:

- Test with **two user-owned accounts** (User A vs User B).
- Test **User A's token** against a **User B-owned object ID** (BOLA/IDOR).
- Test **member role vs admin role** inside an owned organization.
- Test **X-Org-ID / X-Tenant-ID isolation** using owned organizations only.
- Test **old API version vs current API version** for access differences.
- Verify whether **frontend blocks an action but API allows it**.
- Check if a **200 response contains real data** vs only an error body.
- Test **HTTP method override** (e.g., `X-HTTP-Method-Override`).
- Test **batch/graphql endpoints** for excessive data exposure.
- Do **not** test on real third-party data.
